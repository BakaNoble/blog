# 关于我

这个站点主要用来记录一些长期折腾下来的东西：服务器、网络、AI 工具、本地模型、博客建站、OpenWrt、Docker、Kubernetes，以及偶尔出现的动漫、游戏和生活笔记。

我不是专门写教程的人，更像是把自己踩过的坑、查过的资料、实际跑通的配置整理下来。很多内容可能来自一次具体的问题排查，也可能来自某个临时起意的实验。

## 我会写什么

这个博客大概会包含这些内容：

- **网络与路由器折腾**
- **服务器与运维笔记**
- **AI 工具与本地模型**
- **建站与 Web 项目**
- **动漫 / 游戏 / 杂项**

## 这个博客的定位

这里不是一个严格意义上的技术文档站。

更准确地说，它是一个个人知识库：

```text
遇到问题
→ 搜资料
→ 实际测试
→ 跑通或失败
→ 记录下来
```

所以文章里可能会有完整教程，也可能只是某个问题的解决记录；可能有结论，也可能保留一些过程中的判断和取舍。

我更关注“实际是否能用”，而不是写得是否像标准文档。

## 当前技术栈

本站目前使用：

```text
Astro
Fuwari
GitHub
Cloudflare Pages
```

本地写作和修改后，通过 Git 推送到 GitHub，再由 Cloudflare Pages 自动构建和部署。

选择这套方案的原因很简单：静态博客不需要复杂后端，部署成本低，速度快，维护也比较轻。

## 为什么写博客

主要有几个原因。

第一，很多问题解决过一次之后，如果不记录，过段时间还会重新查一遍。

第二，很多工具和配置的教程并不完整，真正跑起来时会遇到各种环境差异。把实际过程写下来，至少以后自己还能翻。

第三，AI 工具越来越多，信息变化很快，单纯收藏链接不够用。把使用体验和判断整理成文章，会更容易沉淀成自己的知识。

最后，也算是给自己留一个长期可维护的内容入口。

## 关于内容准确性

这个博客里的内容主要基于个人实践和当时的环境。

尤其是服务器、网络、AI 工具、模型、平台规则这类内容，变化可能很快。文章里的结论不一定长期有效。

如果你参考这里的配置，建议结合自己的环境重新检查：

```text
系统版本
软件版本
网络环境
平台限制
安全策略
```

我会尽量把测试环境、命令和踩坑点写清楚，但不保证所有内容都能直接复制使用。
